Rolling deployment

This rolls the EC2 instances behind the ELB by increasing the Server count to Max and reducing the Instance count back to Desired
Drawbacks in this approach is that for a small period of time the Website will show inconsistent behavior (i.e Versio 1 and Version 2) during the overlap
Since the ELB will route the requests to both the new and old servers
Termination polciies is set to OldLaunchConfigfirst (so that while reducing count the old instances are terminated)

This is achieved using a AWS Lambda function
